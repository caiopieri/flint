# Contributing to Flint

Thanks for your interest in contributing! Flint is in early development, so the best ways to help right now are feedback, ideas, and discussion.

## How to contribute

### Report bugs
Open an issue with a clear description of the problem, steps to reproduce, and your device/OS version.

### Suggest features
Open an issue with the `enhancement` label. Describe the problem you're trying to solve, not just the solution.

### Submit code
For anything beyond a small fix, open an issue first to discuss the approach — it avoids wasted work on changes that may not fit the roadmap.

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Open a pull request with a clear description of what changed and why

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Write clear commit messages
- If adding new functionality, update documentation accordingly
- Be respectful and constructive in all interactions (see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md))
